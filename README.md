

## Формирование слотов
Слоты формируются автоматически на основе расписания переговорки.

### Выбранный подход
Используется комбинированный подход:
- при создании расписания система заранее генерирует слоты на ближайшие 7 дней;
- при запросе доступных слотов на более поздние даты слоты догенерируются динамически.

  ### Обоснование
Система заранее формирует слоты на ближайшие 7 дней, так как именно этот диапазон покрывает большинство пользовательских запросов. Однако возможны запросы на более дальние даты, поэтому генерация слотов продолжается динамически по мере необходимости.
При этом:
- После каждого прошедшего дня уже сгенерированные слоты частично используются (например, остаётся 6 дней);
- При первом запросе на новую дату за пределами текущего окна система догенерирует недостающие слоты.
Таким образом, пользователь сам неявно сдвигает окно доступных дат своими запросами.

Я думаю что этот подход предотвращает резкий рост нагрузки на базу данных, потому что:

-  Система не хранит слоты на длительный период вперёд, которые возможно не понадобятся, что уменьшает объём хранимых данных.
-  Слоты создаются именно для тех дат, которые реально запрашиваются пользователями.
-  Несмотря на ограниченное окно предгенерации, система корректно обрабатывает запросы на любые даты за счёт генерации слотов по запросу.

### Принцип работы
- Администратор создаёт расписание переговорки.
- После создания расписания:
  - генерируются слоты на ближайшие 7 дней с шагом 30 минут.
- При запросе слотов:
  - если слоты уже существуют, они возвращаются из базы;
  - если запрошенная дата выходит за пределы сгенерированного диапазона слоты создаются.
